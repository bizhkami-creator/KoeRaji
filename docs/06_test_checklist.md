# 06_テストチェックリスト (Test Checklist)

## Day 1-6 基本機能
- [x] アプリが正常に起動すること
- [x] フォルダ選択・再帰的ファイル検索ができる
- [x] 番組情報（局・番組名・日付）が正しくパースされる
- [x] 音声ファイルの再生・一時停止・停止ができる
- [x] 各操作に対してTTS（音声読み上げ）のフィードバックがある
- [x] 検索欄によるリアルタイム絞り込みが動く

## Day 7 検索強化 & 音声操作
- [x] マイク権限のリクエストが表示され、許可できる
- [x] 「再生」「停止」などの音声コマンドが機能する
- [x] 複数キーワード（例：「TBS 荻上」）でAND検索ができる
- [x] 「と」区切りの入力（例：「TBSと荻上」）でAND検索ができる
- [x] 「6月15日」や「6/15」などの日付形式で正しく検索できる
- [x] 全角英数字（例：６月１０日）でも正しく検索できる
- [x] 「〜を探して」という音声コマンドで複数条件の絞り込みができる

## Day 8 自動選択機能
- [x] 検索結果が1件のときに自動で選択される
- [x] 自動選択時に番組情報が読み上げられる
- [x] 検索結果が0件のときに選択が解除される
- [x] 検索欄が空（全件表示）のときは自動選択されない
- [x] 音声検索から自動選択→「再生」コマンドの連携ができる

## Day 9 「○○を再生」コマンド
- [x] 「○○を再生」という音声コマンドで検索が実行される
- [x] 検索結果が1件の場合、そのまま自動再生される
- [x] 検索結果が複数件の場合、最初のファイルが自動再生される
- [x] 検索結果が複数件の場合、件数の案内が読み上げられる
- [x] 検索結果が0件の場合、エラー案内が読み上げられる
- [x] 再生対象に合わせて「選択中の番組」表示が更新される
- [x] 検索語が空の「を再生」は通常の再生として動作する

## Day 9 音声操作の高度化
- [x] 音声認識開始時に再生が一時停止する
- [x] 検索系コマンドのTTS応答後、再生が自動再開する
- [x] 再生系コマンド後は自動再開されない（新しい再生が優先される）
- [x] 音声認識エラー後も再生が自動再開する
- [x] TTS読み上げが終わるまで再生が再開されない

## Day 9 音声操作の自然言語対応
- [x] 「○○を検索」「○○検索」で検索ができる
- [x] 「○○を調べて」「○○調べて」で検索ができる
- [x] 「○○を探す」で検索ができる
- [x] 「○○を再生」が検索コマンドより優先して動作する

## Day 9 自然言語日付検索
- [x] 「おととい」が「と」で分割されずに正しく 2 日前として検索される
- [x] 「3日前」などの数値指定で検索ができる
- [x] 「先週の火曜日」で前週の火曜日の日付が算出される
- [x] 日付表現と一般キーワードの混在（例：おととい 伊集院）でAND検索ができる
